2.1.	Rede Virtual Privada

A Rede Virtual Privada, popularmente chamada de “VPN” é uma abreviatura para Virtual Private Network (Rede Privada Virtual), que se define como: Rede Privada Pessoal, que é construída sobre a infraestrutura de uma rede pública. 
Esta rede privada é uma forma de estabelecer uma conexão com dois computadores através de uma rede pública como se fosse naturalmente uma rede privada com protocolos de segurança e técnicas de tunelamento que dificultam interceptações por terceiros não autorizados nesta rede. (NASCIMENTO, Rafael, 2009, p. 25).
Uma das vantagens desta ferramenta é o fato do custo acessível em que os links dedicados, além de como a internet hoje tornou-se mundialmente acessível em várias áreas do mundo, é possível se conectar a outra ponta desta conexão virtual privada de modo remoto de qualquer lugar que tenha acesso a internet.

2.2.	Componentes da VPN

Uma rede VPN é formada geralmente por um conjunto de tecnologias, algumas das principais são:

2.2.1.	Tunelamento

Um dos componentes da VPN é o tunelamento que é uma técnica que possibilita encapsular um protocolo dentro de outro protocolo qualquer, possibilitando a utilização desta técnica para que seja possível que um pacote dentro de uma rede que não usa o protocolo de endereçamentos IP possa transportar dados através da Internet por um caminho seguro em uma rede não confiável, criando um efeito de túnel na rede. (MARLETA, 2007)

2.2.2.	Criptografia de Dados

A criptografia é um dos itens mais importantes em uma rede VPN, baseada deste os jogos infantis de desvendar enigmas ou símbolos, que de maneira lúdica permite a criança entender aquela simbologia se houver uma tabela relacionada.

O objetivo da criptografia é embaraçar uma sequência de dados para que não seja entendida por aqueles que não fazem parte da VPN, ou seja, apenas o verdadeiro destinatário será capaz de entender os dados originais, pois ele conterá o desembaraçador da criptografia, que é chamado de descriptografia, o efeito reverso da criptografia. (HOYER et al., 2012).
O processo de criptografia de dados e informações funciona através de um conjunto de regras que são aplicadas sobre os dados embaralhando-os, utilizando uma chave padrão de criptografia e basicamente existem dois tipos de chaves que podem ser utilizados para criptografar os dados que são: a chave simétrica e a chave assimétrica que explicaremos a seguir:

•	Chave Simétrica

Também conhecida como chave privada. É uma técnica de criptografia onde a mesma chave é utilizada para criptografar e descriptografar os mesmos dados.

•	Chave Assimétrica

Pode ser chamada de chave pública. É a técnica onde as chaves utilizadas são diferentes, mas relacionadas para criptografar e descriptografar.

2.2.3.	Protocolos de Segurança de Criptografia

Além de chaves de criptografia, temos também protocolos de criptografia que são diferentes das chaves. Esses protocolos fornecem conexões seguras, permitindo que haja uma conexão ponto a ponto com privacidade e integridade dos dados. (IBM, on-line)

•	AES

O protocolo AES é conhecido como Advanced Encryption Standard, ou Padrão de Criptografia Avançada na linguagem traduzida para o Português do Brasil. Este protocolo veio para substituir o DES e o triplo DES onde ainda utiliza substituições e permutações de seus antecessores. Essa criptografia avançada tem por requisito uma segurança mais forte para projetos simples que apresenta um desempenho razoável. (MARTINI, Igor, 2017).

•	SSL

O termo SSL significa Secure Sockets Layer, que para o português traduz como Soquete de Camada de Segurança. Esse protocolo permite que vários serviços sejam tunelados através desse protocolo seguro. A Camada de Soquete Seguro (SSL) da Rede Virtual Privada é uma das técnicas que podem ser usadas com Protocolo de Transferência de Hipertexto (HTTP) de um navegador WEB padrão. O SSL fornece a confidencialidade e a integridade para que o tráfego da VPN viaje através da rede pública (CISCO, 2018, on-line).

•	TLS

Sucedendo o protocolo SSL, o termo TLS significa Transport Layer Security, que traduzido ao idioma do Português de Brasil, significa Camada de Transporte de Segurança. Este protocolo é totalmente baseado no seu anterior TLS, porém de maneira mais aprimorada e com objetivos relacionados a segurança na criptografia, interoperabilidade em quesitos de independência para desenvoltura de novas aplicações, flexibilidade que permite a incorporação de uma nova chave e métodos de criptografia evitando a necessidade de implementar uma nova biblioteca com protocolos de segurança. (SOUZA, Adilson; 2008, p. 28).

2.2.4.	Protocolos da Camada de Transporte de Dados.

A Camada de Transporte de Dados em Redes de Computadores podem ser definidas em dois protocolos de transporte padrão: o TCP (Transmission Control Protocol) e o UDP (User Datagram Protocol).

•	TCP

O TCP (Transmission Control Protocol), em uma definição traduzida livre para o Português, é compreendido como Controle de Transmissão de Protocolo, que realiza o controle de envio e endereçamento de pacotes de informações de dados de internet, ele é responsável também por confirmar se todos os dados foram enviados e recebidos corretamente além de ordená-los. Ao citar a responsabilidade em ordenação de pacotes, aduzimos que o protocolo numera todos os protocolos que serão enviados a fim de conferi-los no recebimento e requisitar o pacote que ficou obstante ou se perdeu durante a transmissão, sabendo de suas eficiências o protocolo pode ser definido com a característica forte de confiabilidade. (CISCO, on-line).

•	UPD

O UDP (User Datagram Protocol), que significa Protocolo de Datagrama do Usuário é coloquialmente traduzido dessa forma ao português do Brasil e ainda coloquialmente chamado de “irmão” do TCP, por fazer o mesmo trabalho que toma como base o envio de pacotes de informações, mas se abstêm de conferir a integridade dos pacotes que foram transitados pela rede, pelo o fato da sua metodologia ser agilidade no envio desses pacotes, entendendo-se que a conferencia de informações e a solicitação de pacotes ausentes e/ou não entregues contribuem para demora. Embora esse método de funcionamento potencialize a ocorrência de erros, ele garante uma comunicação rápida entre dois computadores. (CISCO, on-line).

2.2.5.	Autenticação das Extremidades

A autenticação é um fator de segurança imprescindível, onde é possível utilizar Servidores de Autenticação, Autorização e Contabilização, mais conhecido como AAA, que são uteis para reforçar a segurança na VPN. 
Cada fator tem sua função, sendo: i) Autenticação que identifica o usuário logado para que possa entrar / sair no sistema. ii) Autorização que determina o que cada usuário pode fazer enquanto estiver logado. iii) Contabilização que determina e limita as funções e permissões do usuário enquanto estiver on-line. (CISCO, 2005, on-line).
Todas as mensagens que trafegam na VPN devem ser autenticadas para segurança na emissão de informação dos usuários para que sejam validados através de protocolos de autenticação que são implementados por algoritmos, que se durante o percurso de transporte dessa informação na camada pública de internet for notado alguma alteração nessas informações, o pacote da mensagem inteiro é descartado. Por mais que a mensagem seja criptografada, ela precisa ser autenticada para que exista mais reforço ao fato de prevenção de ataques cibernéticos (GENDORF, 2006).

2.3.	Protocolos da VPN

Os protocolos estabelecem padrões e padrões são necessários para estabelecer qualidades. Também podemos definir protocolos como métodos de alguma função, que no caso é a VPN. Os protocolos de VPN são necessários para construção dessa rede privada e atualmente no mercado existem diversos tipos de protocolos para se realizar o tunelamento.

2.3.1.	Protocolo de Túnel Ponto a Ponto (PPTP)

Como um dos primeiros participantes no mundo dos protocolos de VPN criado, o PPTP tem uma história rica e variada. Ele existe desde os dias do Windows 95, mas conta com o desatualizado pacote de autenticação MS-CHAP v2, o que significa que é fácil de decifrar.

Para um fácil entendimento, define-se esse protocolo assim:

O PPTP (Point-to-Point Protocol- Protocolo de Tunelamento Ponto-a-Ponto) é um protocolo da camada dois do TCP/IP, ou camada de enlace. Criado por um fórum de empresas, foi um dos primeiros protocolos de VPN que surgiram, tinha como objetivo facilitar o acesso de computadores remotos a uma rede privada. É baseado nos protocolos de autenticação do PPP como CHAP (Challenge-handshake Authentication Protocol), MS-CHAP (Microsoft Challenge-handshake Authentication Protocol) -desenvolvido pelo Microsoft-, e PAP (Password Authentication Protocol), o mais inseguro dos protocolos de autenticação citados (AUGUSTO, et al., 2019).

Este protocolo traz consigo uma vulnerabilidade inerente e uma vantagem que é a falta de recursos de criptografia e autenticação, que significa que o PPTP é o protocolo VPN mais rápido. Isso também significa que o conteúdo da sua conexão pode ser visto pelo seu provedor de Internet, sua operadora de conexão a Internet e Organizações Governamentais de Vigilância.

Protocolo de Túnel Ponto a Ponto (PPTP)

2.3.2.	Protocolo de Encaminhado de Camada 2 (L2F)

É um protocolo desenvolvido pela CISCO e atualmente é um protocolo de tunelamento mais antigo do que o L2TP, onde não depende de protocolos de endereçamento IP, permitindo trabalhar com redes ATM e Frame-Relay que não suportam criptografias, o que torna os dados tráfegos vulneráveis a ataques cibernéticos (BALLESTEROS, 2008).

2.3.3.	Protocolo de Túnel de Camada 2 (L2TP) 

Esse protocolo de Tunelamento na Camada 2, reúne as melhores características dos protocolos existentes PPTP e L2F, com um passo significativo em relação ao Protocolo PPTP, este protocolo oferece melhor segurança ao custo de uma redução na velocidade e o protocolo L2TP geralmente é emparelhado com o protocolo IPsec para fornecer criptografia AES-256 a fim de corrigir vulnerabilidades e a combinação dos dois é conhecida como L2TP / IPsec (BORGES; et al., n.p).
Este protocolo foi desenvolvido para suportar dois tipos de tunelamento que é o voluntário que é iniciado por um host remoto que permite o usuário a conectar-se a qualquer provedor de acesso, sabendo que o provedor não cria nenhum tunelamento, esse pode percorrer a vários servidores sem contar com uma específica configuração por ser um protocolo flexível e o segundo tipo de tunelamento suportado é o compulsório que é criado e iniciado automaticamente pelo servidor de acesso à rede, necessitando de uma pré-configuração para determinar o tunelamento baseado nas informações de autenticação de usuário. (BORGES; et al., n.p).

2.5.	Serviço de Rede Cliente – Servidor

Existem quatros tipos de serviço de rede cliente-servidor e RIOS (2012), nos ajuda a criar um dicionário de definições desses serviços e explicá-los, que são: 

•	Gateway: ou “porta de entrada”, é um computador intermediário ou um dispositivo dedicado, responsável por fornecer determinados tipos de serviços.
•	DHCP: É a abreviatura de Dynamic Host Configuration Protocol ou Protocolo de configuração dinâmica de endereços de rede. É um serviço intensamente utilizado para atualizar as configurações de rede nos dispositivos que estejam utilizando o protocolo TCP/IP.
•	Proxy: Ao acessar um computador em que a página da internet solicitada está bloqueada, normalmente, o proxy é o responsável por não permitir o acesso. O Proxy pode realizar bloqueios de páginas na WEB, além de várias funcionalidades. Esse serviço funciona de forma intermediária entre o usuário e a internet. Outra importante função do proxy é manter uma área de acesso rápido às informações já acessadas por outros usuários, evitando a retransmissão das mesmas informações pela rede.
•	Firewall: ou “Muro antichamas” é o serviço responsável por aplicar uma política de segurança nas informações que trafegam na rede. Ele é responsável por bloquear qualquer tentativa de acesso ao seu computador sem autorização. Através do firewall é possível bloquear portas de programas, IPs etc.

Na loja Oficiais de Aplicativos Microsoft Store da Microsoft Corporation, o WiFi Analyzer pode ajudá-lo a identificar problemas de Wi-Fi, encontrar o melhor canal ou o melhor lugar para seu roteador/ponto de acesso com base na frequência do sinal que pode ser aferido como Mega-hertz (MHz) ou Giga-hertz (GHz), transformando seu computador/laptop, tablet ou dispositivo móvel em um analisador para sua rede sem fio. Está ferramenta também está disponível para Sistemas Mobiles, como Android e IOS. (Microsoft Store, on-line).

2.6.5.	Sistema Operacional Windows 10

É um Sistema Operacional desenvolvido pela Microsoft, cujo lançado em 2015, que cada semestre é atualizado a sua versão para uma nova. A versão mais recente liberada a nível global é a 21H1, que a própria desenvolvedora explica que se trata versão do Windows 10 do ano de 2021 sendo do primeiro semestre. Existe a versão para usuário doméstico a qual pode ser utilizada para a presente pesquisa, entretanto, a mais indicada é a PRO, pois é ideal para pequenas empresas ou usuários que necessitam de funcionalidade avançada. (MICROSOFT; 2021, on-line).
